# sanchos-os v0.1 target

## Scope

The `v0.1` milestone is the first version that should feel like a real system layer rather than a loose scaffold.

## Required outcomes

- clean Debian 12 bootstrap path
- Plasma desktop defaults
- KVM/libvirt host stack
- `sanchosctl` as the local control plane
- first-boot flow
- control center shell with useful tabs
- reset path for test machines
- NekoBox integration as the default desktop VPN client

## Command layer goals

`v0.1` should support:

- system info and doctor
- profile enumeration and apply
- module enumeration and enable
- VM create, list, start, stop, delete and snapshot operations
- reset of the bootstrap state on disposable test machines

## UX goals

The user should be able to:

- install a profile on a clean Debian system
- log into a branded Plasma session
- launch the control center
- see host and network state
- create and manage a VM
- launch NekoBox without extra setup

## Not in v0.1

- custom installer ISO
- clustered virtualization
- advanced storage UI
- high-availability features
- custom package repository infrastructure
