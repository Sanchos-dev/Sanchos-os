# sanchos-os v0.1

## Goal

Deliver the first usable prototype of `sanchos-os` on top of Debian 12.

The focus of `v0.1` is not feature volume. The focus is a clean base, a working desktop, a working virtualization stack and a system layout that can grow without turning messy.

## Scope

### Included
- Debian 12 base
- KDE Plasma desktop
- SDDM login manager
- PipeWire audio stack
- KVM, QEMU, libvirt and OVMF
- Podman and Distrobox
- first `sanchosctl` implementation
- `desktop-virt` profile
- first bootstrap flow
- baseline branding hooks and repository layout

### Explicitly out of scope
- clustering
- high availability
- live migration
- distributed storage
- custom installer
- custom package manager
- fully featured graphical control center

## Deliverables

### 1. Bootstrap path
A clean Debian 12 installation can be turned into a `sanchos-os` prototype through `bootstrap/install.sh`.

### 2. Profile system
At least these profiles exist as manifests:
- `desktop`
- `desktop-virt`
- `dev`
- `server-lite`

### 3. CLI control plane
`packages/sanchosctl/` contains a first working command-line tool capable of:
- showing system information
- listing profiles
- listing modules
- checking virtualization support

### 4. Virtualization baseline
The `desktop-virt` profile installs and enables:
- KVM
- QEMU
- libvirt daemon and tools
- UEFI firmware support
- basic desktop companion tooling

### 5. Container baseline
The platform exposes:
- Podman
- Distrobox

## Acceptance criteria

`v0.1` is considered real when the following path works on a test machine or VM:

1. install clean Debian 12
2. run bootstrap
3. log into Plasma
4. verify virtualization support with `sanchosctl system doctor`
5. create or import a VM using the installed stack
6. confirm container tooling is available

## Notes

This release should be boring in the right way. Stable base, sensible defaults, clean repo layout and no fake complexity.
