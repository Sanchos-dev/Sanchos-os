# sanchos-os v0.1 target

## Goal

Deliver the first usable prototype that can turn a clean Debian 12 installation into a recognizable sanchos-os workstation with working desktop and virtualization foundations.

## Included in v0.1

- Debian 12 base path
- KDE Plasma desktop session
- SDDM login manager
- `desktop-virt` profile
- working `bootstrap/install.sh`
- working `sanchosctl` commands for inspection and profile/module actions
- KVM, QEMU and libvirt stack
- Podman and Distrobox
- profile and module manifests stored under `/etc/sanchos-os`
- baseline docs for packaging, modules and VM architecture

## Not included in v0.1

- custom ISO installer
- custom desktop environment
- clustered virtualization
- HA or migration features
- full graphical control center
- opinionated personal service coupling by default

## Success criteria

v0.1 is successful if the following flow works on a clean test VM:

1. bootstrap completes without manual package hunting
2. Plasma session starts cleanly
3. `sanchosctl system doctor` produces useful results
4. `sanchosctl profile info desktop-virt` matches the installed role
5. `sanchosctl module enable virtualization` and `containers` are repeatable
6. a test VM can be created and booted with libvirt tooling

## Deliverables

- repository scaffold
- bootstrap scripts
- first package source for `sanchosctl`
- tested profile manifests
- module manifests with package/service boundaries
