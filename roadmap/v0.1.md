# v0.1 target

The first milestone is no longer only about proving that bootstrap works. It now needs to feel like an actual sanchos-os session.

## Must-have scope

- Debian 12 base
- KDE Plasma desktop
- Plasma + i3 tiling session path
- SDDM branding with purple tones
- NekoBox installed by default for desktop profiles
- `sanchosctl` with system, VM, wallpaper and visual commands
- control center for host, VM and appearance tasks
- first-boot flow that writes and applies wallpaper and visual state
- KVM/QEMU/libvirt integration
- Podman and Distrobox integration
- live-build path for ISO generation

## Visual target for v9

- warm purple default wallpaper (`purple/purple0.png`)
- floating top panel
- rounded and softer surfaces where Plasma supports them
- monochrome-friendly icon theme scaffold
- desktop that feels personal rather than stock Debian

## Still intentionally out of scope

- clustered virtualization
- HA
- a fully custom desktop environment
- a complete custom icon pack with original glyph coverage
- a fully branded graphical installer stack on the same level as mature distros

## Current direction after v9

The next meaningful milestone after this is to make the ISO feel like the installed system on first boot, with tighter live-build branding and a more native control center.
