# sanchos-os v0.1

## Goal

Deliver the first usable prototype of `sanchos-os` on top of Debian 12.

The point of `v0.1` is to establish a clean and testable base: a daily-driver desktop, a working local VM stack, a usable control CLI, and the first service-aware defaults.

## Scope

### Included
- Debian 12 base
- KDE Plasma desktop
- SDDM login manager
- PipeWire audio stack
- KVM, QEMU, libvirt and OVMF
- Podman and Distrobox
- first `sanchosctl` implementation
- `desktop-virt` profile
- bootstrap path from clean Debian
- baseline branding and repository layout
- NekoBox preinstalled on desktop-oriented profiles

### Explicitly out of scope
- clustering
- high availability
- live migration
- distributed storage
- custom installer
- custom package manager
- full graphical control center

## Deliverables

### 1. Bootstrap path
A clean Debian 12 installation can be converted into a `sanchos-os` prototype through `bootstrap/install.sh`.

### 2. Profile system
At least these profiles exist as manifests:
- `desktop`
- `desktop-virt`
- `dev`
- `server-lite`

### 3. CLI control plane
`packages/sanchosctl/` contains a working command-line tool capable of:
- showing system information
- listing and inspecting profiles
- listing and inspecting modules
- checking virtualization support
- applying profile package sets
- basic VM inspection and lifecycle actions through `virsh`

### 4. Virtualization baseline
The `desktop-virt` profile installs and enables:
- KVM
- QEMU
- libvirt daemon and tools
- UEFI firmware support
- bridge helpers
- desktop companion tooling

### 5. Container baseline
The platform exposes:
- Podman
- Distrobox

### 6. Desktop VPN baseline
Desktop-oriented profiles expose:
- NekoBox as the default GUI VPN client
- a reproducible install path through `scripts/install-nekobox.sh`

## Acceptance criteria

`v0.1` is real when the following path works on a test machine or VM:

1. install clean Debian 12
2. run bootstrap
3. log into Plasma
4. verify virtualization support with `sanchosctl system doctor`
5. start NekoBox from the application menu
6. create or import a VM with the installed stack
7. confirm container tooling is available

## Notes

This release should be intentionally narrow. It does not need fake platform scale. It needs to feel coherent and repeatable.
